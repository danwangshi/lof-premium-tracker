# M1 基础设施层 详细设计

## 一、技术选型总览

| 组件 | 选型 | 内存 | 选型理由 | 备选方案及不选原因 |
|------|------|------|---------|-------------------|
| Web 框架 | FastAPI + Uvicorn | ~50MB | async 原生，自动 OpenAPI 文档，Pydantic 请求校验 | Flask（sync 需要 gevent，多 worker 内存翻倍） |
| ORM | SQLAlchemy 2.0 async | ~15MB | 模型定义清晰，查询可读性好，session 管理方便 | 直接 asyncpg（SQL 拼接维护成本高，2C2G 查询量不需要极致性能） |
| 数据库驱动 | asyncpg | ~5MB | 最快的 async PG 驱动，SQLAlchemy async 底层用它 | psycopg2（sync，会阻塞事件循环） |
| 数据库 | PostgreSQL 14 | ~192MB | JSONB 支持，金融数据可靠性，分区表支持 | MySQL（JSONB 支持弱，分区功能不如 PG） |
| 缓存 + 队列 | Redis 7 | ~128MB | TTL 机制天然适合缓存，Streams 提供持久化队列 | 内存 dict（进程重启丢失，无法做队列） |
| 定时任务 | APScheduler | ~5MB | 进程内调度，和 FastAPI 共享事件循环 | Celery（需要额外 broker，2C2G 杀鸡用牛刀） |
| HTTP 客户端 | httpx | ~5MB | async 原生，连接池，HTTP/2 支持 | aiohttp（API 不如 httpx 简洁） |
| 反代 | Nginx | ~2MB | gzip + SSL + 静态文件 + 限流 | Caddy（自动 SSL 但内存占用更高） |
| 公式解析 | Python ast 模块 | 0 | 内置，零依赖，安全可控 | 第三方表达式库（安全控制不如白名单 AST） |
| 日志 | Python logging | ~1MB | 内置，支持 handler 分文件分级别 | structlog（多一个依赖，2C2G 不需要） |

**内存合计：** ~651MB / 2048MB（余量 1.4GB）

**技术选型原则：**
- 能用内置模块就不引入第三方（ast、logging、csv）
- 能单进程解决就不引入分布式组件（APScheduler 不用 Celery）
- 能用简单方案就不追求极致性能（SQLAlchemy ORM 不用 raw SQL）

- 使用 `pydantic-settings`（Pydantic v2 内置），不用 `python-dotenv` 手动加载

## 三、database.py（数据库连接）

### 3.1 实现要点

```python
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession, async_sessionmaker
from sqlalchemy import event, text
from config import Settings

settings = Settings()

engine = create_async_engine(
    settings.DATABASE_URL,
    pool_size=settings.DB_POOL_SIZE,          # 5
    max_overflow=settings.DB_MAX_OVERFLOW,    # 5
    pool_pre_ping=True,                       # 自动检测断连重连

    echo=settings.DEBUG,                      # DEBUG 模式打印 SQL

async_session = async_sessionmaker(
    engine,
    class_=AsyncSession,
    expire_on_commit=False  # commit 后仍可访问对象属性
)
```

### 3.2 依赖注入

```python
async def get_db():
    """FastAPI Depends 注入，自动 commit/rollback"""
    async with async_session() as session:
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()
            raise
```

### 3.3 会话级查询超时

```python
@event.listens_for(engine.sync_engine, "connect")
def set_statement_timeout(dbapi_conn, connection_record):
    """每个新连接设置查询超时"""
    cursor = dbapi_conn.cursor()
    cursor.execute("SET statement_timeout = '5s'")
    cursor.close()
```

### 3.4 JSON 序列化兜底

```python
import json
from datetime import datetime, date
from decimal import Decimal

def json_serializer(obj):
    """FastAPI JSONResponse 的 default 参数"""
    if isinstance(obj, (datetime, date)):
        return obj.isoformat()
    if isinstance(obj, Decimal):
        return float(obj)
    raise TypeError(f"Type {type(obj)} not JSON serializable")
```

### 3.5 注意事项

- `pool_pre_ping=True` 会在每次取连接时发一个 `SELECT 1` 探活，有微小性能开销但防断连
- `expire_on_commit=False` 很重要：commit 后 ORM 对象仍可访问属性，否则会触发懒加载（在 async 下会报错）
- 连接池上限 = pool_size + max_overflow = 10，PostgreSQL max_connections = 20，留 10 个给 migration 和手动连接
- lifespan 中调用 `await engine.dispose()` 关闭连接池，否则进程退出时可能有连接泄漏

### 4.2 关键字段注意事项

**fund_daily：**
- `nav_type` 取值：`confirmed`（确认净值）/ `estimated`（QDII 延迟用昨日净值）
- `data_source` 记录数据来源：`push2` / `akshare` / `tencent` / `manual`

**user_formulas：**
- `version` 字段用于乐观锁，每次 UPDATE version + 1
- `name` 唯一约束在应用层校验（同一用户内唯一），不在 DB 层加 UNIQUE（因为是 per-user 唯一不是全局唯一）

**user_alerts：**
- `condition` 用 JSONB 存储结构化条件：`{"op": "and", "conditions": [{"field": "premium_rate", "op": ">", "value": 5}]}`

**asset_daily：**
- 按月分区：`PARTITION BY RANGE (trade_date)`
- 分区命名：`asset_daily_202601`、`asset_daily_202602`
- 每月 1 日自动创建下月分区

### 4.3 全局规范

- 所有 TIMESTAMP 字段用 `TIMESTAMP WITH TIME ZONE`（Python: `DateTime(timezone=True)`）
- 所有金额字段用 `NUMERIC`（不用 FLOAT，避免精度丢失）
- 所有有更新的表有 `updated_at` 字段，应用层设置 `datetime.now(timezone.utc)`
- ORM 模型的 `__tablename__` 和实际表名一致，不用复数形式

**日期格式统一：** `{date}` 格式为 `YYYYMMDD`（如 `20260529`），不用 `2026-05-29`。

### 5.2 实现要点

```python
import redis.asyncio as aioredis
import json
from typing import Optional, Any

_pool: aioredis.Redis = None

async def init_redis(url: str, max_connections: int = 20):
    global _pool
    _pool = aioredis.from_url(
        url,
        decode_responses=True,    # 自动 decode bytes → str
        max_connections=max_connections,
    )

async def cache_get(key: str) -> Optional[Any]:
    """读缓存，Redis 不可用返回 None"""
    try:
        raw = await _pool.get(key)
        return json.loads(raw) if raw else None
    except Exception:
        return None  # 降级

async def cache_set(key: str, data: Any, ttl: int):
    """写缓存，Redis 不可用跳过"""
    try:
        await _pool.set(key, json.dumps(data, ensure_ascii=False, default=str), ex=ttl)
    except Exception:
        pass  # 降级，不抛异常

async def cache_delete(key: str):
    try:
        await _pool.delete(key)
    except Exception:
        pass

async def cache_delete_pattern(pattern: str):
    """批量删除匹配的 key，用 SCAN 而非 KEYS（避免阻塞）"""
    try:
        async for key in _pool.scan_iter(match=pattern, count=100):
            await _pool.delete(key)
    except Exception:
        pass

async def is_redis_available() -> bool:
    try:
        await _pool.ping()
        return True
    except Exception:
        return False

async def close_redis():
    await _pool.close()
```

### 5.3 注意事项

- 用 `scan_iter` 而非 `keys` 命令删除匹配 key：`KEYS *` 在 key 数量大时会阻塞 Redis
- `decode_responses=True` 让所有返回自动 decode，不用手动 `.decode()`
- `default=str` 在 json.dumps 中处理 Decimal/datetime 等不可序列化类型
- 所有操作 try-except，不抛异常到调用方——Redis 是缓存层，不可用时降级读 DB
- `_pool` 全局单例，lifespan 中创建和关闭

### 5.4 部分数据防护

实时行情采集时，如果 push2 返回的数据量 < 上次的 80%，不覆盖 rt:all：

```python
async def safe_set_realtime(data: dict, threshold: float = 0.8):
    current_count = len(data)
    cached = await cache_get("rt:all")
    cached_count = len(cached) if cached else 0

    if cached_count > 0 and current_count < cached_count * threshold:
        logger.warning(f"数据不完整: {current_count}/{cached_count}，保留旧缓存")
        return False  # 不覆盖

    await cache_set("rt:all", data, ttl=60)
    return True
```

## 六、mq.py（Redis Streams 消息队列）

### 6.1 事件类型表

| type | 触发源 | payload | 消费者处理 | 刷新物化视图 |
|------|--------|---------|-----------|------------|
| realtime | fetcher/realtime | 全量实时行情 | 写 Redis rt:all + rt:close:{date} | 否 |
| nav | fetcher/fundamental | 净值+申赎 | 写 Redis nav:all | 否 |
| kline | fetcher/historical | 日线 K 线 | 写 Redis kline:fund:{date} | 否 |
| info | fetcher/info | 持仓+基础信息 | 写 DB + Redis info:{code} | 否 |
| daily_save | scheduler | 标记 is_final | 合并→计算→写 DB | 是 |

### 6.2 实现要点

```python
STREAM_KEY = "stream:events"
GROUP_NAME = "pipeline_processor"
CONSUMER_NAME = "worker-1"

async def init_consumer_group():
    """创建消费者组（已存在则跳过）"""
    try:
        await _pool.xgroup_create(STREAM_KEY, GROUP_NAME, id="0", mkstream=True)
    except redis.ResponseError as e:
        if "BUSYGROUP" not in str(e):
            raise

async def publish_event(event_type: str, payload: dict) -> str:
    """生产者：发布事件，返回消息 ID"""
    try:
        msg_id = await _pool.xadd(STREAM_KEY, {
            "type": event_type,
            "data": json.dumps(payload, ensure_ascii=False, default=str),
            "ts": datetime.now(timezone.utc).isoformat(),
        })
        return msg_id
    except Exception:
        logger.error(f"MQ publish failed: type={event_type}")
        return ""  # 降级，不抛异常

async def consume_events(count: int = 10, block_ms: int = 2000) -> list[dict]:
    """消费者：读取事件"""
    try:
        results = await _pool.xreadgroup(
            GROUP_NAME, CONSUMER_NAME,
            streams={STREAM_KEY: ">"},
            count=count, block=block_ms,
        )
        events = []
        for stream_name, messages in results:
            for msg_id, fields in messages:
                events.append({
                    "id": msg_id,
                    "type": fields["type"],
                    "data": json.loads(fields["data"]),
                    "ts": fields.get("ts"),
                })
        return events
    except Exception:
        return []  # 降级

async def ack_event(msg_id: str):
    """确认消费"""
    try:
        await _pool.xack(STREAM_KEY, GROUP_NAME, msg_id)
    except Exception:
        pass

async def get_stream_length() -> int:
    """监控：Stream 长度"""
    try:
        info = await _pool.xinfo_stream(STREAM_KEY)
        return info["length"]
    except Exception:
        return -1
```

### 6.3 注意事项

- `mkstream=True`：如果 Stream 不存在自动创建，避免首次运行报错
- `BUSYGROUP` 错误：消费者组已存在时忽略，不抛异常
- `block=2000`：阻塞等待 2 秒，没有事件时返回空列表，不浪费 CPU
- `xadd` 的 `maxlen` 参数可选：如果 Stream 积压超过 10000 条自动裁剪（防内存泄漏）
- 消费者失败不 ack，下次重启时重新消费——但要处理重复消费的幂等性（消费者内部检查数据是否已处理）

## 七、metrics.py（监控指标）

### 7.1 指标清单（12 个）

| stream_lag | gauge | mq | Stream 未消费事件数 |
| memory_mb | gauge | system | 进程内存（psutil.Process().memory_info().rss / 1024²） |

| asset_count | gauge | DB | asset_master 行数 |
| redis_memory_usage_pct | gauge | Redis | Redis 内存使用率（每 5 分钟，>80% 告警） |
| partition_days_remaining | gauge | DB | 距最近分区边界的天数（<7 天告警） |

### 7.2 核心函数

```python
from dataclasses import dataclass, field
from datetime import datetime

@dataclass
class Metrics:
    fetch_status: dict = field(default_factory=dict)  # {source: {success, fail, last_ok, last_fail, duration_ms}}
    api_requests: int = 0
    db_query_durations: list = field(default_factory=list)  # 近 100 次查询耗时
    cache_hits: int = 0
    cache_misses: int = 0

    def record_fetch(self, source: str, success: bool, duration_ms: float, business_error: bool = False):
        if source not in self.fetch_status:
            self.fetch_status[source] = {"success": 0, "fail": 0, "business_error": 0, "last_ok": None, "last_fail": None, "duration_ms": 0}
        s = self.fetch_status[source]
        if success:
            s["success"] += 1
            s["last_ok"] = datetime.now(timezone.utc).isoformat()
        else:
            s["fail"] += 1
            s["last_fail"] = datetime.now(timezone.utc).isoformat()
        if business_error:
            s["business_error"] += 1
        s["duration_ms"] = duration_ms

    def record_api_request(self):
        self.api_requests += 1

    def record_db_query(self, duration_ms: float):
        self.db_query_durations.append(duration_ms)
        if len(self.db_query_durations) > 100:
            self.db_query_durations = self.db_query_durations[-100:]

    def get_metrics(self) -> dict:
        avg_db = sum(self.db_query_durations) / len(self.db_query_durations) if self.db_query_durations else 0
        total_cache = self.cache_hits + self.cache_misses
        cache_rate = self.cache_hits / total_cache * 100 if total_cache > 0 else 0
        return {
            "fetch_status": self.fetch_status,
            "api_requests": self.api_requests,
            "db_query_avg_ms": round(avg_db, 2),
            "cache_hit_rate": round(cache_rate, 1),
        }

metrics = Metrics()  # 全局单例
```

### 7.3 告警函数

```python
import httpx

_alert_cooldown: dict = {}  # {source: last_alert_time}

async def alert(title: str, message: str, level: str = "P1"):
    """发送告警到企业微信 webhook"""
    if not settings.ALERT_WEBHOOK_URL:
        return

    # 同一 source 同一小时不重复告警
    cooldown_key = f"{title}:{level}"
    now = datetime.now(timezone.utc)
    if cooldown_key in _alert_cooldown:
        if (now - _alert_cooldown[cooldown_key]).total_seconds() < 3600:
            return

    _alert_cooldown[cooldown_key] = now

    try:
        async with httpx.AsyncClient() as client:
            await client.post(settings.ALERT_WEBHOOK_URL, json={
                "msgtype": "markdown",
                "markdown": {
                    "content": f"## [{level}] {title}\n{message}\n时间: {now.strftime('%Y-%m-%d %H:%M:%S')}"
                }
            }, timeout=10)
    except Exception:
        logger.error(f"告警发送失败: {title}")
```

### 7.4 自愈规则（消费者中执行）

async def load_calendar(pool):
    """启动时加载当年+明年日历到内存"""
    global _calendar, _calendar_loaded
    try:
        rows = await pool.fetch(

def is_trading_day(date=None) -> bool:
    if not _calendar_loaded:
        return False  # 宁可不采也不乱采
    target = date or datetime.now().date()
    return _calendar.get(target, False)

def get_latest_trading_date() -> date:
    today = datetime.now().date()
    for d in range(0, 10):
        check = today - timedelta(days=d)
        if _calendar.get(check, False):
            return check
    return today  # fallback

async def ensure_partition(conn, table: str, target_date: date):
    """确保目标日期所在月的分区存在"""
    partition_name = f"{table}_{target_date.strftime('%Y%m')}"
    start_date = target_date.replace(day=1)
    if start_date.month == 12:
        end_date = start_date.replace(year=start_date.year + 1, month=1)
    else:
        end_date = start_date.replace(month=start_date.month + 1)

    exists = await conn.fetchval(
        "SELECT EXISTS(SELECT 1 FROM pg_tables WHERE tablename = $1)", partition_name
    )
    if not exists:
        await conn.execute(f"""
            CREATE TABLE IF NOT EXISTS {partition_name}
            PARTITION OF {table}
            FOR VALUES FROM ('{start_date}') TO ('{end_date}')
        """)
        logger.info(f"分区创建: {partition_name}")

## 十一、httpx 全局客户端

### 11.1 实现

```python
import httpx

_http_client: httpx.AsyncClient = None

async def init_http_client():
    global _http_client
    _http_client = httpx.AsyncClient(
        http2=True,
        timeout=httpx.Timeout(connect=10, read=30, write=10, pool=10),
        limits=httpx.Limits(
            max_connections=20,
            max_keepalive_connections=10,
            keepalive_expiry=30,
        ),
    )

async def close_http_client():
    await _http_client.aclose()

def get_http_client() -> httpx.AsyncClient:
    return _http_client
```

### 11.2 注意事项

- `http2=True`：HTTP/2 多路复用，减少 TCP 连接数
- `timeout` 四个维度分别设置：connect（建连）/ read（读响应）/ write（发请求）/ pool（等待连接）
- `max_connections=20`：连接池上限，匹配 Redis 和 DB 的连接数
- `keepalive_expiry=30`：空闲连接 30 秒后关闭，释放资源
- 所有 fetcher 共享这个 client，不在各处 `async with httpx.AsyncClient()`
- lifespan 中创建，shutdown 时 `aclose()`

## 十三、错误现场快照

### 13.1 实现

```python
import psutil
import traceback
import json
from pathlib import Path

SNAPSHOT_DIR = Path("/var/log/jinkuaicha/snapshots")
MAX_SNAPSHOTS = 50

async def save_snapshot(request_id: str, request_params: dict, exc: Exception):
    """50000 异常时自动保存现场"""
    try:
        snapshot = {
            "request_id": request_id,
            "timestamp": datetime.now(timezone.utc).isoformat(),
            "error": str(exc),
            "traceback": traceback.format_exc(),
            "request_params": request_params,
            "memory_mb": round(psutil.Process().memory_info().rss / 1024 / 1024, 1),
        }
        filepath = SNAPSHOT_DIR / f"{datetime.now().strftime('%Y%m%d_%H%M%S')}_{request_id}.json"
        filepath.write_text(json.dumps(snapshot, ensure_ascii=False, indent=2, default=str))

        # 清理旧快照
        snapshots = sorted(SNAPSHOT_DIR.glob("*.json"))
        while len(snapshots) > MAX_SNAPSHOTS:
            snapshots.pop(0).unlink()
    except Exception:
        logger.error("快照保存失败")
```

### 13.2 安全注意事项

- 快照中不保存 Authorization header、密码、token 等敏感信息
- `request_params` 只保存 query params 和 path params，不保存 request body（可能含密码）
- 快照文件权限 600，仅 deploy 用户可读
- 50000 异常时才触发，4xx 错误不保存

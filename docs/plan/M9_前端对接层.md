## 三、js/api-client-v1.js

### 3.1 基础配置

- 基础 URL：/api/v1
- 请求拦截：自动加 Authorization header（从 Supabase session 获取 token）
- 响应拦截：统一错误码分发
- 超时：默认 15 秒
- 重试：网络超时自动重试 3 次（指数退避 1s/2s/4s）

### 3.2 请求缓存

客户端维护一个请求缓存 Map：key = URL + 参数序列化，value = {data, expiresAt}。

缓存命中 → 直接返回，不发请求。
缓存过期 → 发请求，更新缓存。
手动刷新 → 跳过缓存。

用途：基金列表 → 详情 → 返回列表，列表数据直接从缓存返回，零延迟。

### 3.3 token 刷新单例

多个请求同时收到 40102 → 只有一个请求执行 token 刷新，其他等待同一个 Promise。

逻辑：
1. 请求 A 收到 40102 → 检查是否有 refreshPromise
2. 没有 → 创建 refreshPromise（调 Supabase 刷新 token）
3. 请求 B 也收到 40102 → 检测到 refreshPromise 存在 → await 它
4. 刷新完成 → refreshPromise 置 null → 所有等待的请求用新 token 重试

### 3.4 错误码分发

| 错误码 | 前端动作 |
|--------|----------|
| 40101 | 弹登录框 |
| 40102 | 静默刷新 token，失败弹登录框 |
| 40103 / 40104 / 40105 | 清除 token，弹登录框 |
| 40301 / 40302 | 提示"无权限" |
| 40001 | 高亮具体字段（从 detail 数组取字段名） |
| 40002 | 公式编辑器中标记语法错误位置 |
| 40003 | 公式编辑器中标记非法字段 |
| 40401 ~ 40404 | 展示"资源不存在" |
| 42901 / 42902 | 展示倒计时（从 detail 中取 retry_after） |
| 50001 ~ 50003 | 展示"服务暂时不可用" |

### 3.5 API 方法封装

按模块封装，不在业务代码中直接调用 fetch：

基金：getFunds / getFundDetail / getFundBatch / getFundChart / getFundHoldings / exportFunds
资产：getAssets / getAssetDetail / getAssetChart / exportAssets
数据：queryDaily / batchQuery
自选：getWatchlist / addWatchlist / removeWatchlist
预警：getAlerts / createAlert / deleteAlert
公式：getFormulas / createFormula / updateFormula / deleteFormula / validateFormula
公式组：getFormulaGroups / createFormulaGroup / evalFormulaGroup
系统：getHealth / getMonitor / triggerRefresh

## 四、js/sse-client.js

### 4.1 连接管理

- 交易时段（通过后端响应的 data_type 字段或本地时间判断）自动建立 SSE 连接
- 非交易时段不连接，不重连

### 4.2 增量更新

SSE 推送的是增量 changes（只包含变化的基金和字段），前端直接用 changes 更新对应单元格，不做全量对比。

首次数据通过 GET /api/v1/funds 获取完整列表。

### 4.3 重连策略（jitter）

基础间隔：1 秒 → 2 秒 → 4 秒 → 8 秒 → 最大 30 秒

加随机抖动：实际间隔 = 基础间隔 + random(0, 基础间隔 × 0.5)

目的：服务端重启时，100 个客户端的重连分散在 30-45 秒窗口内，不会同时涌入。

### 4.4 页面可见性控制

监听 document.visibilitychange 事件：

| 状态 | 行为 |
|------|------|
| 可见 | 正常接收 SSE + 更新 UI |
| 不可见 | 暂停 SSE 消息处理（连接保持），暂停 UI 更新 |
| 重新可见 | 立即请求最新数据（补偿不可见期间变化），恢复 SSE 处理 |

### 4.5 非交易时段

SSE 连接断开后不自动重连。页面数据保持最后收到的 SSE 数据。用户手动刷新时走 GET /api/v1/funds，响应中 data_type=closing 告诉前端切换到收盘数据模式。次日交易时段自动重新建立连接。

## 五、js/filter-panel.js

### 5.1 筛选条件

| 条件 | 类型 | 说明 |
|------|------|------|
| premium_min / premium_max | 数值范围 | 溢价率范围 |
| amount_min / amount_max | 数值范围 | 成交额范围 |
| fund_type | 下拉选择 | ETF / LOF / QDII / 全部 |
| purchase_status | 下拉选择 | open / suspended / 全部 |
| turnover_min | 数值 | 最低换手率 |
| volume_ratio_min | 数值 | 最低量比 |
| search | 文本输入 | 代码或名称模糊搜索 |

### 5.2 搜索防抖

输入 300ms 防抖。输入 < 2 个字符不发请求。

### 5.3 筛选预设

保存到 localStorage，支持多个预设：
- 预设 1：高溢价 ETF（premium_min=5, fund_type=ETF）
- 预设 2：可申购 QDII（fund_type=QDII, purchase_status=open）
- 默认预设：全部（无筛选条件）

一键切换预设。

## 七、js/formula-editor.js

### 7.1 语法高亮

| 元素 | 颜色 | 示例 |
|------|------|------|
| 字段名 | 绿色 | close, nav, premium_rate |
| 运算符 | 灰色 | +, -, *, /, >, < |
| 函数名 | 蓝色 | abs, max, min, round |
| 数字常量 | 橙色 | 100, 0.5 |
| 三元表达式 | - | ? 和 : |

### 7.2 错误标记

校验失败时，在错误位置加红色下划线。后端返回的 detail 中如果包含位置信息，在对应字符位置标记。

### 7.3 常用示例

输入框下方显示可点击的示例：
- close / nav - 1（收盘溢价率）
- premium_rate > 5 ? premium_rate : 0（筛选高溢价）
- abs(premium_rate - premium_3d)（溢价率变化幅度）

点击直接填入输入框。

### 7.4 字段名自动补全

用户输入 "pre" → 弹出候选项 premium_rate / premium_3d → 按 Tab 填入。

数据源：/api/v1/formula-fields 返回的字段列表。

### 7.5 实时预览

输入公式后，用前 5 只基金的数据调用前端 evaluator 计算，在编辑器下方展示结果。用户立刻看到公式是否符合预期。

### 7.6 校验防抖

输入 300ms 防抖后才调用 /api/v1/formulas/validate。短表达式（< 3 个字符）不发请求，直接用前端 parser 校验。

## 八、js/virtual-scroll.js

### 8.1 原理

只渲染可视区域内的表格行（约 20-30 行）。1,500 只基金的数据全部在内存中，但 DOM 节点只有 30 个。

滚动时动态替换 DOM 内容（回收离开可视区域的行，创建进入可视区域的行）。

### 8.2 性能

- 1,500 行 × 15 列 = 22,500 个 DOM 节点（无虚拟滚动）
- 虚拟滚动后：30 行 × 15 列 = 450 个 DOM 节点
- 滚动流畅度显著提升

### 8.3 与列管理器的配合

虚拟滚动的行结构需要和列管理器（column-manager.js）的列配置同步。列隐藏/显示/排序变化时，虚拟滚动需要重新计算行宽度和渲染内容。

## 十一、三态 UI

每个数据展示区域有三种状态：

| 状态 | UI | 触发条件 |
|------|------|----------|
| loading | 骨架屏（灰色占位条模拟表格行） | 请求发出后，数据返回前 |
| empty | "暂无数据" + 建议操作 | data=[] |
| error | 错误提示 + 重试按钮 | API 返回错误 |

骨架屏比 spinner 体感更快（用户觉得数据在加载中，而不是"卡了"）。

---

## 十二、data_type 处理

后端响应 meta 中的 data_type 字段决定前端显示模式：

| data_type | 前端行为 |
|-----------|----------|
| realtime | 显示实时价格列、实时溢价率列、SSE 增量更新 |
| closing | 隐藏实时列，显示"收盘数据"标签 |
| realtime_available=false | 显示"实时数据暂不可用"提示，隐藏实时列 |

## 十四、预警条件编辑器（P1）

### 14.1 UI

```
条件 1: [字段下拉] [比较符下拉] [数值输入]    [AND/OR 下拉]
条件 2: [字段下拉] [比较符下拉] [数值输入]
```

```
  "conditions": [

}
```

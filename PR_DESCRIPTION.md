# PR 描述（GitHub 提交用）

## 🎯 简要说明

本 PR 合并 danwangshi 分支的协作开发成果，新增**场内份额追踪**、**申购限额筛选**、**365天历史图表**等核心功能。

---

## ✨ 主要变更

### 1️⃣ 场内份额数据集成
- 集成上交所/深交所官方 API 获取 LOF 基金场内份额
- 每日 07:00 自动抓取 T-1 日数据（APScheduler 定时任务）
- 计算并展示"新增份额"变化
- 前端表格和移动端卡片均显示份额数据（单位：万份）

### 2️⃣ 申购限额多选筛选
- 移除"暂停申购基金自动过滤"逻辑
- 新增下拉多选框：支持"暂停申购"、"开放申购"、具体限额
- 一键清空筛选功能
- 智能优先级：暂停申购 > 开放申购 > 具体数值

### 3️⃣ 365天历史图表
- 历史数据保留期从 21 天扩展到 365 天
- 支持七日/一月/三月/六月/一年切换
- 指标体系：价格+净值 / 溢价率（正红负绿）
- 热门基金图表预渲染缓存（每 5 分钟刷新）

### 4️⃣ 其他改进
- 数据源架构优化（AkShare 主源 + Legacy 备源 + 熔断机制）
- 文档体系完善（README、CHANGELOG、ENV_SETUP）
- 代码结构优化（requirements.txt 移至根目录、数据源模块化）

---

## 🔌 新增 API

| 端点 | 说明 |
|------|------|
| `GET /api/purchase-limits` | 获取申购限额选项 |
| `GET /api/shares` | 获取份额数据 |
| `POST /api/shares/fetch` | 触发份额抓取 |
| `GET /api/tasks` | 查看后台任务状态 |

## 📊 数据库变更

**新增字段：**
- `premium_snapshots.shares` - 场内份额

- `premium_snapshots.shares_source` - 数据来源
- `premium_snapshots.shares_incr` - 新增份额

**新增表：**
- `daily_kline` - 365天日线数据存储
